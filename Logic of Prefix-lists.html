The Logic of Prefix-list

How do prefix-lists work

Posted by TuanICT on Dec 4, 2019

The first thing you have to know is Prefix-list's usage is the same as access-list, Prefix-list is tied to Distribute list in order to filtering unnecessary routes.

Let's divide the problem into 4 scenarios:
- Match exact routes.
- Match routes using GE only.
- Match routes LE only.
- Match routes using GE and LE.

Match exact routes
Ip prefix-list MyList permit 10.11.12.0/24
This means the first 24 bits match exact 10.11.12 and the subnet mask must be exactly: 24.
Rewrite into access-list: Ip access-list 101 permit 10.11.12.0 0.0.0.255

Match routes using GE only.
Condition: prefix-len < ge < ge
Ip prefix-list MyList permit 10.11.12.0/24 ge 25
This means the first 24 bits match exact 10.11.12 and the subnet mask must be [25,32]
So 10.11.12.13/24 won't match this statement and will be filtered.

Match routes using LE only.
Condition: prefix-len < ge < ge
Ip prefix-list MyList permit 10.11.12.0/24 le 30
This means the first 24 bits match exact 10.11.12 and the subnet mask must be [24,30]


Match routes using LE and GE.
Condition: prefix-len < ge < ge
Ip prefix-list MyList permit 10.11.12.0/24 ge 30 le 32
This means the first 24 bits match exact 10.11.12 and the subnet mask must be [30,32]


More scenarios
Condition: prefix-len < ge < ge
Ip prefix-list MyList permit 0.0.0.0/0 le 32
This means permit everything
